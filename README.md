## ⭐ 功能特性

- **多源图库**：支持 4 大图库导入方式 (从其他插件一键同步 & 图床同步 & 群聊偷图 & 手动传图)，支持自动去重，支持可视化的图库管理。
- **自动图像特征标注**：上传图片后，调用 LLM 自动生成多个特征标签，辅助插件检索调用。
- **图像智能匹配**：基于 LLM 分析用户意图和对话语义，从图库中找出最符合需求的图片。
- **群聊智能斗图**：支持加入重复图片队形、按概率连发相关图片，以及在连续图片团战中快速语义分析并检索相似图库图片。
- **全新可视化 Pages**：提供功能强大的内嵌 "Plugin UI Page" WebUI 管理中心，支持图库上传、批量筛选、标签编辑、进度查看、数据备份与恢复等，完整替代原生插件配置 WebUI。
- **备份恢复**：支持通过 Plugin UI Page 进行插件数据的导入/导出，支持自动定时备份。
- **自定义文案**：自定义各种发图场景下的回复文案，支持固定文案或 LLM 接管回复。

## 📱 在聊天中使用插件

本插件目前支持 3 种由 LLM 驱动或标签检索驱动的表情包回复能力：

### 1. 为用户寻找表情包

   在唤醒 bot (@机器人、使用唤醒词唤醒 bot、回复 bot、私聊等场景) 的对话中，包含特定关键词 ("表情包"、"照片" 等)，即可要求 bot 在图库里检索合适的图像，例如：

   ```
   @机器人 发一张开心的表情包
   @机器人 有没有可爱的美少女照片
   来一张 <bot 唤醒词> 的图片
   ```

   插件会根据用户描述，从图库中选择满足用户要求的图片发送。

### 2. 对话中主动发送表情包：

   开启该功能后，bot 可以使用 LLM 分析自己发送的对话文本，并基于文本语义和图像标签，从图库里选取符合对话语境的表情包，发送给聊天用户。

### 3. 群聊智能斗图：

   开启该功能后，bot 会在群聊中观察短时间窗口内的图片对话，进行以下 3 种 “斗图” 行为：

   - **加入队形**：如果多名群友连续发送相同图片，bot 会跟着队形进行接龙。
   - **图片连发**：bot 有一定概率，在某次图片回复中，连续发送多张表情包。
   - **参与团战**：如果群聊进入斗图团战，bot 会抽取其中 2 张图片快速分析语义，从自己的图库中搜索表情包参与团战。

### 5. 插件自动偷图

本插件支持从指定的 QQ 群中自动偷图，支持自动去重、人工&机器审核。工作流如下：

```
插件从指定 QQ 群自动偷图 -> 将图像载入图片池 -> 筛选后启动自动标签 -> 图像正式入库
```

具体而言：

1. 插件从群聊中收集的图片，首先进入 [**待筛选图片池**]，此时不会被 LLM 打标签，也不会参与本插件的图库检索。
2. 用户或机器人，需要在 UI Page 中进行批量筛选，决定 [**待筛选图片池**] 中 丢弃 / 保留 哪些图片。
3. 保留的图片，会进入 [**固化图像库**]，按 "手动上传的图片" 相同的规则，参与打标签和检索。

<img src="https://raw.githubusercontent.com/QingchenWait/astrbot_plugin_smart_imagechat_hub/refs/heads/main/imgs/filter_auto_collected_images.png" width="100%" style="border: 2px solid gray;"/>

点击 UI Page [**插件能力配置**] 栏中的 [**自动收集群友发的图**] 按钮，即可进行偷图行为的配置。

| 配置项                               | 说明                                                         | 默认值  |
| ------------------------------------ | ------------------------------------------------------------ | ------- |
| 启用自动收集图片功能                 | 开启后，插件会在指定 QQ 群中自动收集图片，放入 [待筛选图片池]。 | `true`  |
| 自动将所有待筛选图片入库             | 关闭时，用户可手动指定 [待筛选图片池] 中哪些图片存入 [固化图像库]。开启后，图像将实时自动入库。<br />***注意：开启本功能，可能存在无效表情包过多、使图像库爆满等风险。因此，除非群聊可信任，否则不建议开启。*** | `false` |
| 将入库图像加入插件其他功能的筛选流程 | 开启后，[固化图像库] 中的图片，会和手动上传图像一起，作为 “为用户寻找表情包” 和 “对话中主动发送表情包” 等插件图片功能的检索候选项。 | `true`  |
| 收集来源 QQ 群群号列表               | 每行一个 QQ 群号。机器人只在这些群聊中自动收集图片。         | `[]`    |
| 单张图片尺寸上限                     | 图像大小的上限。以 `KB` 作为计量单位。                       | `1024`  |
| 非表情包图像过滤策略                 | 控制自动收集时如何处理照片、截图等非表情包图像：<br /> (1) `none` / “不过滤” ：去重后的全部图像，都会进入待筛选池；<br /> (2) `loose` / “宽松过滤 (保留照片等普通图像)”：沿用旧版文件头尺寸/比例过滤，跳过明显截图；<br /> (3) `strict` / “严格过滤 (只保留表情包)” ：只收 OneBot/NapCat raw message 中明确带表情包元数据的图片。 | `loose` |
| 待筛选图片池存储数量上限             | 设置为 `-1` 表示无上限。                                     | `30`    |
| 不触发收集的 QQ 号 (黑名单)          | 每行一个黑名单 QQ 号。插件在任何群聊中，都不会收集由这个黑名单里的 QQ 号发送的图片。 | `[]`    |
| 自动拒绝曾经被丢弃过的图片           | 开启后，会记录在用户操作 [待筛选图片池] 时，被丢弃的图片信息；再次见到相同图片时直接跳过。 | `true`  |
| 固化图像库存储数量上限               | 设置为 `-1` 表示无上限。<br />***注意：如果开启了 [自动将所有待筛选图片入库] 功能，建议设置一个数量上限，以避免图库占用过多内存。*** | `300`   |

### 6. 从其他插件直接导入表情包

本插件支持直接从 AstrBot 服务端安装的**其他插件**中，将图库**一键导入本插件**。支持增量更新、自动去重等。

本功能仅限 Plugin UI Page 中使用。工作流如下：

```
选择插件图库 -> 将图像载入图片池 -> 筛选后启动自动标签 -> 图像正式入库
```

具体而言：

1. 用户点击 UI Page [**插件能力配置**] 栏中的 [**从其他插件导入图库**] 按钮，插件会显示 AstrBot 安装的所有插件的持久化数据目录。<br /> 选择目标目录 (支持选择子目录)，开始导入图像库。
2. 插件从群聊中收集的图片，首先进入 [**导入进程**] 图片池，此时不会被 LLM 打标签，也不会参与本插件的图库检索。
3. 用户需要在 UI Page 中进行批量筛选，决定 [**导入进程**] 图片池中是否要丢弃图片、取消导入进程。点击 "开始" 启动自动标签流程。
4. 保留的图片，会进入 [**来自外部插件的图像库**]，按 "手动上传的图片" 与自动收集流程中 "固化图像库" 相同的规则，参与打标签和检索。

<img src="https://raw.githubusercontent.com/QingchenWait/astrbot_plugin_smart_imagechat_hub/refs/heads/main/imgs/import_image_from_other_plugins.png" width="100%" style="border: 2px solid gray;"/>

一些可选项如下：

| 配置项                 | 配置位置    | 说明                                                     |
| --------------------- | ---------- | -------------------------------------------------------- |
| 将图像的上一级目录名称，<br /> 作为标签之一导入 | `选择外部图库目录` 窗口 | 有一些 AstrBot 的插件，是按照**子目录**分类表情包的 (例如 `meme_manager`)。<br /> 因此，勾选这个选项，可以直接让本插件继承一些原始插件的分类标签，作为图像的特征标签之一。|
| 统计图库规模          | `选择外部图库目录` 窗口 | 用户在选择了目录以后，需要首先点击左下角的 [**统计数据规模**] 按钮，确认该目录中包含多少张图片。<br /> 统计完毕后，即可点击右下角的 [**开始导入**] 按钮，开始数据同步。
| 批量丢弃              | `导入进程` 栏目 | 不让哪些图片进入本插件的图像库。<br /> **注意**：丢弃过一次的图片，就不会再被本插件识别并导入。 |
| 取消导入              | `导入进程` 栏目 | 停止自动标签进程，取消后会释放所有未打标签图像。<br /> 取消以后，可以重新进行一次导入流程，插件 **不会将它们直接丢弃**。<br /> 对于导入了一半的图像库，插件支持**自动去重**。|

### 7. 从图床导入表情包

本插件支持从**图床**中，通过**单向同步**将图库一键导入本插件。支持增量更新、自动去重等。

本功能仅限 Plugin UI Page 中使用，目前支持 `Cloudflare R2` (海内外均可正常访问)。工作流如下：

```
配置图床 API -> 将图像同步进图片池 -> 筛选后启动自动标签 -> 图像正式入库
```

图床配置方式，可以参考：[使用 Cloudflare R2 搭建个人/团队专属“永久”图床](https://involutionhell.com/zh/docs/learn/cs/dev-tips/cloudflare-r2-sharex-free-image-hosting)

## ❓ 常见问题

**Q：上传图片后为什么没有自动生成标签？**

A：请确认已在 AstrBot 系统配置中设置了**默认图片转述模型**。可以在 UI Page 的进度面板中查看是否有报错信息。

**Q：标签生成失败了怎么办？**

A：可以在 UI Page 中点击"刷新"按钮重新触发标签生成，或检查图片转述模型是否正常工作。

**Q：AstrBot 版本号 `<4.24.2` 的用户，是否可以使用本插件？**

A：可以使用本插件中的**标准配置页面**，页面中包含了能使插件正常运行的全部功能。但是，标准配置页面无法实现数据备份、图库缩略图预览等功能，因此，我们建议您升级 AstrBot 版本。

**Q：如何只隐藏部分图片而不删除？**

A：在插件配置的 `hidden_images` 中添加图片的相对路径，图片将不会参与检索但保留在磁盘中。

**Q：开启“发送表情包时统一样式 (GIF)”后，原图会被改成 GIF 吗？**

A：不会。插件只会在发送前生成临时 1 帧 GIF 副本，原图库文件、标签、备份和后续处理仍保持原格式。如果运行环境缺少 Pillow 或转换失败，会自动发送原图。

**Q：自定义回复文案支持哪些变量？**

A：支持以下占位符：

- `{filename}` - 匹配图片的文件名
- `{tags}` - 图片的特征标签（顿号分隔）
- `{need}` - 用户需求摘要
- `{reason}` - 匹配理由

**Q：如何备份和迁移图库？**

A：使用 UI Page 中的"导出配置"功能，会生成包含所有数据的 ZIP 文件。在新环境中使用"导入配置"即可恢复。
